# 機能要件定義書

## 1. プロジェクト管理

| 機能 | 説明 |
|------|------|
| 新規作成 | 空のプロジェクトを生成 |
| 保存 | SQLiteファイルとして保存（Ctrl+S） |
| 読込 | SQLiteファイルから復元（Ctrl+O） |
| エクスポート | JSON形式で出力 |
| インポート | JSON形式から読み込み |

プロジェクトファイル拡張子: `.lifecanvas`（実体はSQLite）

---

## 2. 家族

### データ項目

| フィールド | 型 | 説明 |
|-----------|-----|------|
| name | str | 名前 |
| relation | enum | husband / wife / child / other |
| birth_date | date | 生年月日 |
| annual_income | int | 額面年収（円） |
| bonus | int | ボーナス（円） |
| salary_growth_rate | float | 昇給率（%/年） |
| retirement_age | int | 定年年齢 |
| pension_start_age | int | 年金受給開始年齢 |
| occupation | enum | 会社員 / 公務員 / 自営業 / パート / 無職 |

### 制約

- 人数制限なし
- 子供は`birth_date`が未来の場合、シミュレーション途中で誕生扱い
- 妻の就業ワークフローは`income_modifiers`で定義（育休・パート・復職）

---

## 3. ライフイベント

### 対応イベント

| イベント | 影響 |
|---------|------|
| 結婚 | 一時支出 |
| 出産 | 給付金・一時支出 |
| 育休 | 年収変更・給付金 |
| 車購入 | CarPlanと連動 |
| 住宅購入 | HousePlanと連動 |
| リフォーム | 一時支出 |
| 住宅売却 | 一時収入・ローン精算 |
| 賃貸運用 | 継続収入 |
| 定年 | 年収→0、年金開始 |
| 相続 | 一時収入 |

### 仕様

- 自由追加可能
- 各イベントは `elapsed_year`（経過年数）で発生時期を指定
- `one_time_cost` / `one_time_income` で一時的な収支を定義
- `details` dictで関連プランへの参照を保持

## 4. 住宅

### データ項目

| フィールド | 型 | 説明 |
|-----------|-----|------|
| purchase_price | int | 購入価格（円） |
| location | str | 所在地 |
| purchase_year | int | 購入年（経過年数） |
| loan | LoanPlan | ローン設定 |
| maintenance_cost | int | 修繕費（10年周期） |
| annual_property_tax | int | 固定資産税（年額） |
| annual_fire_insurance | int | 火災保険（年額） |

### LoanPlan

| フィールド | 型 | 説明 |
|-----------|-----|------|
| borrowed_amount | int | 借入額（円） |
| term_years | int | 借入期間（年） |
| interest_rate | float | 金利（%） |
| loan_type | enum | fixed / variable |
| repayment_method | enum | equal_payment / equal_principal |
| interest_adjustments | dict | 金利推移シナリオ {年: 金利%} |

### 対応機能

- 固定金利 / 変動金利
- 金利推移シナリオ（例: 毎年0.2%上昇、最大3%）
- 繰上返済（一括・一部）
- 売却（一時収入としてキャッシュ化）
- 旧居の賃貸運用（年間手残り設定）
- 住宅ローン控除（残高×0.7%、上限21万円、13年間）

## 5. 投資

### データ項目

| フィールド | 型 | 説明 |
|-----------|-----|------|
| account_type | enum | nisa / ideco / taxable / cash |
| owner | str | 所有者 |
| initial_balance | int | 初期残高 |
| monthly_deposit | int | 毎月積立額 |
| annual_return_rate | float | 想定年利（%） |
| changes_schedule | dict | 変更スケジュール |

### 仕様

- NISA: 非課税運用
- iDeCo: 所得控除あり（税金計算に反映）
- 特定口座: 利益に20.315%課税
- 現金: 年利0%の積立（元本保証）
- `changes_schedule`で途中からの積立額変更に対応

## 9. 税金・社会保険

### 計算対象

| 項目 | ロジック |
|------|---------|
| 所得税 | 累進税率（給与所得控除→課税所得→速算表） |
| 住民税 | 課税所得×10% + 均等割5,000円 |
| 社会保険料 | 額面年収×15%（簡易） |
| 住宅ローン控除 | 残高×0.7%（上限21万円、最大13年） |
| iDeCo控除 | 拠出額を所得控除に反映 |

## 11. シミュレーションエンジン

### 年次ループ処理フロー

```
for year in range(simulation_years):
    1. 家族の年齢更新
    2. 収入算出（年収・年金・給付金）
    3. 税金・社会保険の計算
    4. 住宅ローン控除の適用
    5. ライフイベントの処理
    6. 住宅費用（ローン返済・固定資産税・修繕）
    7. 教育費
    8. 車費用
    9. 保険料
   10. 生活費
   11. 投資積立（複利計算）
   12. キャッシュフロー集計
   13. 現金不足時の投資取崩し
   14. 純資産算出
```

## 12. AI分析

| 分析項目 | 判定条件 |
|---------|---------|
| 赤字年度 | 年間収支 < 0 |
| 現金不足 | 現預金残高 < 0 |
| 教育費ピーク | 教育費が最大となる年 |
| 老後資金不足 | 65歳以降の純資産 < 2,000万円 |
| FIRE判定 | 純資産 ≥ 年間支出×25 |
| 金利リスク | 変動金利3%シナリオで赤字転落 |
| 改善提案 | 上記を基にテキスト生成 |

## 15. UI構成

| 画面 | 説明 |
|------|------|
| ダッシュボード | サマリーカード + グラフ + リアルタイム調整スライダー |
| 家族構成 | テーブル形式の編集 |
| 住宅計画 | フォーム形式の入力 |
| 投資計画 | テーブル形式の編集 |
| 自動車計画 | フォーム形式の入力 |
| 保険計画 | テーブル形式の編集 |
| シナリオ比較 | 比較グラフ + テーブル |
| レポート出力 | PDF/Excelボタン |

### 操作フロー

1. アプリ起動 → サンプルデータでダッシュボード表示
2. サイドバーで各設定画面へ遷移
3. パラメータ変更 →「適用して再計算」→ グラフ即時更新
4. ダッシュボードのスライダーでリアルタイム調整
5. シナリオ保存 → 比較画面で並列表示
6. レポート出力
