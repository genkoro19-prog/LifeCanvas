## フェーズ1: 計算ロジック

- 妻は個人借入、個人支出、本人NISAを先に処理する
- 妻の余剰が家計拠出開始基準以下なら家計拠出を0円にする
- 基準を超えた月は余剰全体を家計負担上限まで拠出する
- 家計残額は夫が負担する
- 夫の最低維持預金と目標預金を別々に管理する
- 現金残高は0円未満にせず、支払えなかった金額を未充足額へ分離する

## フェーズ2: 投資配分

- 夫婦それぞれの基本NISAを本人資金から処理する
- 夫の目標預金を超えた余剰だけを夫NISAへ追加投資する
- 自動追加投資は初期状態でOFFとする
- 妻の基本NISA・家計負担後の余剰は妻預金へ残し、自動的に妻NISAへ移さない
- 夫NISAへ追加できない場合、夫資金から妻NISAへの移転を検討する
- 妻NISAへの資金移転は年間管理上限とその他資金移転額を考慮する
- 妻本人資金によるNISAと夫資金による追加投資を別集計する
- NISA後の余剰資金の配分先を設定データとして保持する
- NISAの年間・生涯上限は買付元本に適用し、運用益を含む評価額には上限を設けない

## フェーズ3: 個人借入

- `PersonalDebt`モデルを追加した
- 借入名、借入者、月額、残り年数だけで登録できる
- 選択行から借入詳細ダイアログを開ける
- 現在残高、当初元金、金利、返済開始、返済方式、ボーナス返済、支払元、備考を入力・保存できる
- 月次返済を家計拠出・個人支出・NISAより先に処理する
- 指定期間終了後は返済を停止する
- 現在残高と金利がある場合は月次利息と残高を更新する

## フェーズ4: UI改善

- 詳細設定を左カテゴリメニューと右設定ページに分割した
- 各設定カードへカテゴリを明示的に割り当てた
- 各カテゴリを原則2列で配置した
- 外側余白、カード間隔、フォーム行間、入力欄高を圧縮した
- 各カテゴリの3つ目以降の低頻度項目を折りたたみ領域へまとめた
- 折りたたみ時は内容領域を非表示にし、不要な空白を残さない
- 詳細設定上部に固定の「設定を反映して再計算」ボタンを配置した
- かんたん入力に家計ルールと個人借入を追加した
- 旧家計割合の表示を隠し、妻の家計負担上限と新しいプレビューへ置き換えた
- 全画面のスピンボックスとコンボボックスでマウスホイールによる値変更を防止した
- 旧保存データと旧入力部品向けの互換フィールドを維持した

## 注意事項

- 年間110万円は税務上の非課税保証ではなく、アプリ内の資金移転管理上限として扱う
- NISAの1,800万円は買付元本の生涯上限であり、運用益を含む評価額の上限ではない
- 夫の預金は目標預金へ到達後、自動追加投資をONにしている場合は目標付近で維持されやすい
- さらに現金を残す場合は夫の目標預金を引き上げるか、自動追加投資をOFFにする
- NISA後の配分先は設定として保持しており、課税口座や繰上返済の詳細な取引計算は今後の専用機能で拡張可能
- ユーザーが内容と実際の画面を確認するまでPRはドラフトのままとし、mainへはマージしない

## 追加フィードバック: 預金目標とNISA進捗

- 妻の目標預金を追加し、詳細設定・かんたん入力の双方から編集可能にした
- 自動追加投資ON時は、妻の目標預金を超えた部分だけを妻本人NISAへ追加する
- 夫の基本NISA計算前に、前月残高へ月間基本貯金額を積み上げた現金目標を確保する
- 夫の個人支出は最低維持預金を割り込ませない
- 家計・借入などで最低維持預金を割った月数と最大不足額を年別に記録する
- 年別表へ夫婦それぞれのNISA年額、買付元本累計、進捗、評価額を追加した
- NISA買付元本が1/4、1/2、1/1へ到達した年を結果画面と年表へ表示する
