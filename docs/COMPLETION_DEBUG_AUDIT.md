# LifeCanvas 完成前デバッグ監査

## 目的

完成版候補を配布する前に、計算ロジック、入力反映、ボタン発火、保存・読込、表示、PDF、Windowsビルドを一通り確認する。

## 今回確認した主要リスク

### 1. 入力変更が再計算へ届かない

- 文字入力とプルダウンは一部接続済みだった。
- 金額・年齢・金利・年数などの数値入力は接続漏れがあり得た。
- 子ども、住宅、車、収入期間、財布、イベント、個人借入は各エディタの `changed` を同じ再計算経路へ統一した。
- かんたん入力、詳細設定、上部の再計算ボタン、PDF出力前の再計算を同じ経路へ統一した。

### 2. ボタンの二重発火・旧関数発火

- 継承前のメソッドへ接続されたボタンが残る可能性があった。
- 再計算、詳細設定の反映、PDF、かんたん入力の確定は、起動時に現在の監査済みメソッドへ再接続する。

### 3. PDFグラフ欠落

- Base64データURIをHTMLへ埋め込む方式はQt PDFで不安定だった。
- PNGを `QTextDocument` の画像リソースとして直接登録する方式へ変更した。
- PDF出力前に必ず現在の入力を再計算する。

### 4. 個人借入残高

- 返済不足でも残月数だけ減り、未払い残高が消える可能性があった。
- 満額返済した月だけ残月数を減らす。
- 年末の借入残高を純資産から控除する。
- 現在サポートする返済方式・支払元以外は、黙って概算せず入力エラーにする。

### 5. 年別表の「家計不足」

- 夫が正常に負担した家計残額まで「家計不足」へ含めていた。
- 今後は実際に払えなかった未充足額だけを表示する。
