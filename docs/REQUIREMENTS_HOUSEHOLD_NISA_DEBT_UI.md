# LifeCanvas 改修要件

## 家計負担・預金・NISA・個人借入・詳細設定UI

- ステータス: 方針確定・実装前
- 作成日: 2026-07-20
- 対象: Windows版 LifeCanvas
- この文書の目的: 実装AIが迷わず設計・実装・テストできるよう、会話で確定した要件を一つにまとめる

---

## 1. 今回の目的

現在の計算では、妻が出産・育休・休職・退職などで収入を失った場合でも、通常時の家計負担やNISA積立などが残り、妻の預金が不自然に大きなマイナスになる可能性がある。

また、詳細設定画面は項目自体には不足がない一方、すべての設定が縦方向に並び、余白も大きいため、画面が長くなりすぎている。

今回の改修では次を実現する。

1. 妻の家計負担を実際の余剰資金に連動させる
2. 妻の個人支出と本人NISAを家計拠出より優先する
3. 家計の不足分は原則として夫が負担する
4. 夫の預金を生活防衛資金の基準として管理する
5. 目標預金を超えた余剰をNISAへ自動配分する
6. 夫のNISAへ追加投資できなくなった後、条件付きで妻NISAへ資金を移す
7. 奨学金などの個人借入を簡単に登録できるようにする
8. 預金残高を無制限にマイナス表示しない
9. 詳細設定の項目を減らさず、短く・見やすく・探しやすいUIへ再構成する
10. 数値入力や選択欄がマウスホイールで意図せず変わる問題を全画面で解消する

---

## 2. 基本となる財布の考え方

LifeCanvasでは、少なくとも次の資金所有者を区別する。

- 夫の個人資金
- 妻の個人資金
- 家計に使用する資金

ただし、家計の生活防衛資金は夫の預金を基準として管理する。

妻の預金は妻個人の資産として表示・計算するが、家計維持のために自動的かつ無制限に取り崩さない。

### 2.1 資金移動と消費の区別

以下は消費ではなく、財布間または資産間の移動として扱う。

- 夫から家計への資金移動
- 妻から家計への資金移動
- 夫から妻へのNISA資金移転
- 現金からNISAへの投資
- NISA売却による現金化

生活費、住宅費、教育費、個人支出、借入返済などの実支出とは分け、二重計上しない。

## 3. 妻の毎月の資金配分

妻の収入および給付金は、標準では次の順に配分する。

1. 妻の個人借入返済
2. 妻の個人支出
3. 妻本人の基本NISA積立
4. 妻の手元余裕資金
5. 妻の家計拠出

妻の家計負担を先に固定額で引かない。

### 3.1 妻の利用可能収入

```text
妻の利用可能収入
= 妻の給与手取り
+ 妻の給付金
+ 妻のその他収入
```

給与、育児休業給付、出産手当相当、その他給付は内訳を分けて表示する。

### 3.2 妻の余剰資金

```text
妻の余剰資金
= 妻の利用可能収入
- 妻の個人借入返済
- 妻の個人支出
- 妻本人の基本NISA積立
```

資金不足時に基本NISAを妻の預金から継続するかは設定可能にする。

### 3.3 家計拠出開始基準

妻の余剰資金が設定額以下の場合、妻は家計へ拠出しない。

- 設定名: 家計拠出開始基準額
- 初期値案: 月30,000円
- 意味: 家計負担の下限ではなく、妻側に残す毎月の余裕額

```text
妻の余剰資金 <= 家計拠出開始基準額
→ 妻の家計拠出額 = 0円
```

余剰資金が開始基準額を超えた場合は、超過部分だけを拠出候補とする。

```text
妻の家計拠出候補額
= max(0, 妻の余剰資金 - 家計拠出開始基準額)
```

### 3.4 妻の家計負担上限

妻には月額の家計負担上限を設定する。

```text
妻の実際の家計拠出額
= min(
    妻の家計負担上限,
    妻の家計拠出候補額,
    その月の家計必要残額
  )
```

要件:

- 妻の家計負担下限は設けない
- 収入が0円または余剰が基準以下なら、家計負担は0円になり得る
- 育休中などに家計負担を維持するため、妻預金を自動で大きく取り崩さない
- 上限を超えた妻の余剰は、妻預金または妻本人の追加投資へ回す

### 3.5 計算例

設定:

- 妻の個人支出: 月40,000円
- 妻NISA: 月30,000円
- 奨学金返済: 月15,000円
- 家計拠出開始基準: 月30,000円
- 妻の家計負担上限: 月100,000円

妻の利用可能収入が月150,000円の場合:

```text
150,000 - 15,000 - 40,000 - 30,000 = 65,000円
65,000 - 30,000 = 35,000円
妻の家計拠出額 = 35,000円
```

妻の利用可能収入が月100,000円の場合:

```text
100,000 - 15,000 - 40,000 - 30,000 = 15,000円
余剰が30,000円以下のため、妻の家計拠出額 = 0円
```

## 4. 夫の家計負担

夫は家計の主負担者とする。

夫の家計負担には固定の上限・下限を設けない。

```text
夫の実際の家計負担額
= 家計必要額
- 妻の実際の家計拠出額
- その他の家計収入
```

妻の収入が減少すると、夫の家計負担は自動的に増加する。

妻の家計拠出が0円の場合、原則として夫が家計必要額の残りを負担する。

### 4.1 夫だけで家計を維持できない場合

標準の調整順序:

1. 夫NISAへの追加投資を停止
2. 夫資金による妻NISAへの追加投資を停止
3. 夫の目標預金への基本貯金を停止または減額
4. 夫の基本NISAを減額または停止
5. 任意支出を減額
6. 夫の預金を取り崩す
7. 最低維持預金を下回る場合は警告
8. 設定に応じてNISA等を売却
9. それでも不足する場合は未充足額を記録

妻の家計負担を上限以上に強制増額して不足を埋めない。

---

## 5. 夫の預金管理

家計の生活防衛資金は、夫の預金を基準にする。

夫の預金には次の3項目を設定する。

### 5.1 現在預金

シミュレーション開始時点の夫の預金残高。

### 5.2 最低維持預金

緊急時を除き、可能な限り下回らないようにする金額。

夫の預金が最低維持預金未満の場合:

- 追加NISAを行わない
- 妻NISAへの資金移転を行わない
- 預金の回復を優先する
- 基本NISAの減額・停止を検討する

### 5.3 目標預金

通常時に確保しておきたい預金額。

「預金上限」という名称は使用せず、「目標預金」と表示する。

夫の預金が目標預金未満の場合、設定した月間基本貯金額を預金する。

- 設定名: 目標到達までの月間基本貯金額
- 目標預金を超える分までは貯金しない
- 目標到達後の余剰は投資候補とする

## 6. NISAの配分ルール

### 6.1 基本方針

夫婦は、それぞれ自分の収入または自分の既存資金から、自分のNISAを積み立てる。

- 夫の基本NISA
- 妻の基本NISA

妻の基本NISAは、妻の家計拠出より先に計算する。

夫の基本NISAは、家計成立と最低維持預金の確認後に計算する。

### 6.2 夫NISAへの追加投資

次の条件を満たす場合、余剰資金を夫NISAへ追加投資する。

- 夫の預金が目標預金以上
- その月に家計成立後の余剰がある
- 夫の当年利用可能NISA枠が残っている
- 夫の生涯利用可能NISA枠が残っている

目標は、可能な範囲で夫のNISA枠を早く使用すること。

### 6.3 夫NISAへ追加できない場合

夫の年間枠または生涯枠の制約により、夫NISAへ追加できない状態になっても余剰がある場合、妻NISAへの資金移転を検討する。

### 6.4 妻NISAへの夫資金の移転

夫資金を妻名義のNISAへ使用する場合、生活費ではなく、夫から妻への資金移転として記録する。

設定項目:

- 妻への年間資金移転管理上限
- 初期値案: 1,100,000円
- 妻が当年すでに受けたその他資金移転額
- 夫から妻NISAへの当年移転累計
- 年間残り移転管理可能額

```text
妻NISAへの当年追加可能額
= 年間資金移転管理上限
- 妻の登録済みその他資金移転額
- 夫から妻NISAへの当年移転累計
```

要件:

- 管理上限を超える自動資金移転を行わない
- 妻自身の収入・給付・既存預金からのNISA投資は、この夫資金移転累計に含めない
- 家計生活費の夫負担は、妻NISAへの資金移転累計に含めない
- アプリは税務上の非課税を保証せず、計画上の管理額と注意表示にとどめる
- 制度変更へ対応できるよう、上限値は設定可能にする

### 6.5 双方のNISAへ追加できない場合

次の配分先を選択可能にする。

- 夫の課税口座
- 妻の課税口座
- 夫の預金
- 住宅ローンの繰上返済
- その他の目標資金

## 7. 個人借入・奨学金

住宅ローンとは別に、夫婦それぞれの借入を登録できるようにする。

対象例:

- 奨学金
- 自動車ローン
- カードローン
- フリーローン
- 親族からの借入
- 分割払い
- その他の個人債務

### 7.1 かんたん入力

最低限、次だけで登録できること。

- 借入名
- 借入者: 夫 / 妻 / 家計
- 毎月返済額
- 残り年数、または終了年月

入力例:

```text
借入名: 奨学金
借入者: 妻
月額: 15,000円
残り期間: 8年
```

### 7.2 詳細入力

必要に応じて次を設定できる。

- 返済開始年月
- 返済終了年月
- 現在残高
- 元金
- 金利
- 返済方式
- ボーナス返済
- 繰上返済予定
- 支払元
- 備考

元金や金利が不明な場合は、指定期間だけ発生する固定支出として扱う。

### 7.3 計算上の優先度

個人借入返済は、個人支出およびNISAより優先する。

借入者が妻の場合、原則として妻資金から差し引く。

本人資金が不足する場合の支払元は設定可能にする。

- 本人預金
- 配偶者
- 家計
- 未充足額として警告

完済年月以降は、自動的に返済額を0円にする。

---

## 8. 月次計算の標準順序

年単位だけでなく、内部計算は月単位を基本とする。

1. 夫婦それぞれの給与手取りを計算
2. 給付金とその他収入を加算
3. 夫婦それぞれの個人借入返済を計算
4. 妻の個人支出を確保
5. 妻本人の基本NISAを計算
6. 妻の余剰資金を計算
7. 妻の家計拠出開始基準額を確保
8. 妻の家計負担上限の範囲で家計へ拠出
9. 家計必要額の残りを夫が負担
10. 夫の個人支出と個人借入返済を計算
11. 家計が成立するか確認
12. 夫の最低維持預金を確認
13. 夫の目標預金未達時は基本貯金を実行
14. 夫の基本NISAを計算
15. 余剰があれば夫NISAへ追加投資
16. 夫NISAへ追加できない場合、妻NISAへの資金移転を検討
17. 年間資金移転管理上限を確認
18. 残余資金を設定された次順位へ配分
19. 月末の夫預金・妻預金・NISA・借入残高を更新
20. 年間画面では月次結果を合計して表示

---

## 9. 預金が不足した場合

預金残高を無制限にマイナス表示しない。

通常の表示上は、現金残高を原則0円未満にしない。

支払えなかった金額は、次の別項目として記録する。

- 未充足額
- 資金不足額
- 配偶者補填額
- 預金取り崩し額
- NISA売却額

警告例:

- 妻の育休により、妻の家計拠出が月100,000円から0円になった
- 不足分を夫が負担した
- 夫の預金が最低維持預金を下回った
- NISA積立を減額・停止した
- 借入返済を本人資金だけでは維持できなかった
- 生活費を満たせず未充足額が発生した

## 10. 詳細設定UIの再構成

### 10.1 現状の課題

- 詳細設定の項目は十分だが、縦方向へ長く並びすぎている
- グループ間・フォーム行間・説明文周辺の余白が大きい
- 夫と妻の対応する項目が離れており、比較しにくい
- 使用頻度の低い詳細項目も常時表示されている
- 目的の項目を探すために長いスクロールが必要
- 同じ画面の中に複数の大きな編集領域が連続している

### 10.2 基本レイアウト

詳細設定タブを、次の構造へ変更する。

```text
┌──────────────┬──────────────────────────────┐
│ カテゴリ一覧   │ 選択中カテゴリの設定内容       │
│              │                              │
│ 基本情報       │ 2列のコンパクトなカード配置     │
│ 収入・働き方   │                              │
│ 家計・預金     │                              │
│ NISA・投資     │                              │
│ 子ども・教育   │                              │
│ 住宅           │                              │
│ 車             │                              │
│ 借入・イベント │                              │
│ 年金・計算条件 │                              │
└──────────────┴──────────────────────────────┘
```

推奨実装構造:

- 左側: 固定幅のカテゴリメニュー
- 右側: QStackedWidget等によるカテゴリ別ページ
- 右側の各カテゴリページだけをスクロール可能にする
- 詳細設定全体を一つの極端に長いページにしない
- ネストした複数スクロール領域は原則作らない

### 10.3 カテゴリ案

1. 基本情報
   - 開始年月
   - 夫婦の年齢
   - 現在預金
   - 定年・年金の基本条件

2. 収入・働き方
   - 夫の年齢別収入
   - 妻の年齢別収入
   - 出産・育休・時短・復職
   - 給付金

3. 家計・預金
   - 生活費
   - 夫の家計負担
   - 妻の家計拠出開始基準
   - 妻の家計負担上限
   - 夫の最低維持預金
   - 夫の目標預金
   - 基本貯金額

4. NISA・投資
   - 夫の基本NISA
   - 妻の基本NISA
   - 追加投資ルール
   - 妻への年間資金移転管理上限
   - 双方のNISA後の配分先

5. 子ども・教育
   - 子ども
   - 出産時期
   - 教育方針
   - 教育費

6. 住宅
   - 賃貸
   - 住宅ローン
   - 住み替え
   - 売却・賃貸化

7. 車
   - 車両
   - 買い替え
   - 維持費

8. 借入・イベント
   - 奨学金等の個人借入
   - 一時収入
   - 一時支出
   - その他ライフイベント

9. 年金・計算条件
   - 年金開始
   - 税・社会保険の概算条件
   - インフレ率
   - 運用利回り
   - その他の高度な仮定

### 10.4 カテゴリ内の配置

デスクトップ幅を前提として、カテゴリ内は原則2列で配置する。

例:

```text
┌──────── 夫 ────────┐  ┌──────── 妻 ────────┐
│ 現在年収             │  │ 現在年収             │
│ 年齢別収入計画       │  │ 育休・時短・復職      │
│ 個人支出             │  │ 個人支出             │
│ 基本NISA             │  │ 基本NISA             │
└───────────────────┘  └───────────────────┘
```

要件:

- 夫婦の対応項目は左右に並べる
- 住宅と住み替えなど、関連項目を同じページへまとめる
- 一つのカードに無関係な設定を詰め込まない
- 画面幅が足りない場合は1列へ切り替えてよい
- Windows版の最小画面幅でもラベルが切れないこと

### 10.5 余白とサイズの基準

推奨初期値:

- ページ外側余白: 12〜16px
- カード間隔: 8〜12px
- カード内側余白: 10〜12px
- フォーム行の縦間隔: 4〜6px
- フォーム列間隔: 8〜12px
- 通常入力欄の高さ: 30〜34px
- セクション見出し下の余白: 6〜8px

禁止事項:

- 各グループに大きな一律余白を入れない
- 長いページ途中に不要な addStretch() を置かない
- 非表示・無効化した旧入力欄のレイアウト領域を残さない
- 入力欄を必要以上に横幅いっぱいへ伸ばさない
- 単位表示だけのために大きな横幅を消費しない

### 10.6 説明文の整理

項目は減らさないが、説明文による高さを抑える。

- 通常説明は1行以内を基本とする
- 長い説明はヘルプアイコン、ツールチップ、または「説明を見る」へ移す
- 重要な注意だけをインライン表示する
- 同じ内容の説明を複数箇所へ繰り返さない
- 注意文は入力欄の下ではなく、カード上部またはヘルプへ集約する

### 10.7 使用頻度の低い詳細項目

低頻度項目は折りたたみ可能にする。

例:

- 金利や返済方式の詳細
- 高度な税・社会保険仮定
- NISA売却順序
- ボーナス返済
- 住み替えの詳細費用
- その他の高度なシナリオ条件

折りたたんだ際は、余白を残さず高さを詰める。

### 10.8 操作ボタン

- 再計算・適用などの主要操作は常に見つけやすくする
- ページ最下部までスクロールしないと操作できない構造を避ける
- 詳細設定タブ上部または固定フッターに主要操作を置く
- 自動再計算状態は現在の「最新 / 未反映」を維持する

### 10.9 詳細設定UIの受け入れ条件

1. 項目数を減らさずにカテゴリ分割されている
2. 詳細設定タブ全体が一つの長大な縦ページではない
3. 目的のカテゴリへ1回のクリックで移動できる
4. 夫婦の対応項目を同時に比較できる
5. 標準ウィンドウサイズで不要な横スクロールが発生しない
6. 各カテゴリ内の主要項目が概ね1〜2画面以内に収まる
7. 低頻度項目を閉じた際、空白領域が残らない
8. 非表示の旧項目がレイアウト高さを消費しない
9. 説明文による過剰な高さがない
10. 入力欄の高さと行間が画面全体で統一されている
11. 再計算・適用操作を長いスクロールなしで実行できる
12. 現在の入力内容と保存データの互換性を維持する

## 11. マウスホイールによる誤変更防止

アプリ全画面で、入力部品にマウスカーソルが乗った状態でスクロールしても、値を変更しない。

対象:

- QSpinBox
- QDoubleSpinBox
- QComboBox
- 日付・年月選択部品
- 独自の数値入力部品
- その他ホイールで値が変化する入力部品

標準動作:

- マウスホイールは画面スクロールとして扱う
- 入力欄にフォーカスがあっても通常ホイールでは値を変更しない
- キーボードの上下キーは使用可能
- 増減ボタンは使用可能
- コンボボックスを意図的に開いた後の操作は、誤操作にならない範囲で別途制御してよい

各画面で個別対応せず、共通Widgetまたは共通イベントフィルターで一括適用する。

## 13. 最低限必要なテスト

### 13.1 妻の収入減少

- 通常勤務時は上限まで家計拠出できる
- 育休給付のみになったら拠出額が減る
- 余剰が30,000円以下なら拠出額が0円になる
- 妻の家計負担が設定上限を超えない
- 妻預金が家計負担のため無制限にマイナスにならない

### 13.2 夫の家計負担

- 妻の拠出減少分を夫が負担する
- 夫の家計負担に固定上限・下限がなくても計算できる
- 夫の収入不足時に定義した順で投資・貯金を停止できる
- 最低維持預金を下回った際に警告が出る

### 13.3 預金と投資

- 夫の預金が目標未満なら設定額を貯金する
- 目標到達時は目標超過分だけ投資候補になる
- 夫NISAの残存枠へ追加投資される
- 夫NISAへ追加できない場合に妻NISAを検討する
- 妻への年間資金移転管理上限を超えない
- 妻本人資金によるNISAと夫資金によるNISAを区別する

### 13.4 個人借入

- 月額と残り年数だけで登録できる
- 指定期間だけ返済が発生する
- 完済後は返済が0円になる
- 妻の奨学金返済が妻の余剰計算より先に差し引かれる

### 13.5 UI

- 詳細設定の各カテゴリへ直接移動できる
- 低頻度項目の折りたたみで空白が残らない
- 夫婦項目が左右対応して表示される
- 1120x760以上で操作不能なレイアウト崩れがない
- 1480x940で過剰な空白や不必要な長いスクロールがない
- すべての数値入力・選択欄で通常ホイールによる値変更が発生しない
